# 🚨 Scam Reporting

Help protect the community by reporting projects you believe are scams or bad actors.

---

## How to Report

1. Navigate to the project's detail page
2. Scroll to the bottom and click **"Report Project"**
3. Select a reason:
   - Rug pull / Exit scam
   - Honeypot (can't sell)
   - Fake team or plagiarised content
   - Unregistered securities offering
   - Phishing / clone project
   - Other
4. Add a detailed description and evidence (transaction hashes, links, screenshots)
5. Submit — the report is visible to the community and admins

---

## What Happens After You Report

| Stage | Description |
|-------|-------------|
| **Submitted** | Report visible to community immediately |
| **Community review** | Other users can upvote/downvote the report |
| **Admin review** | Team reviews evidence |
| **Confirmed** | Project tagged as "Suspected Scam" or removed |
| **Rejected** | Insufficient evidence — no action taken |

---

## Rewards for Valid Reports

| Report Type | Reward |
|-------------|--------|
| Report confirmed as scam | **100 $SCORE** |
| First to report a major scam | **500 $SCORE** |
| Report leads to recovery info | **Bonus — varies** |

{% hint style="success" %}
Rewards are distributed automatically when an admin confirms the report.
{% endhint %}

## Penalties for False Reports

{% hint style="danger" %}
**Do not abuse the report system.** Filing false reports to attack competitors or legitimate projects is a serious violation.
{% endhint %}

| Violation | Penalty |
|-----------|---------|
| False report | **-50 reputation** |
| Coordinated false report campaign | **Account suspension** |
| Repeated abuse | **Permanent ban** |

---

## Evidence Guidelines

Strong reports include:

| Evidence Type | Example |
|---------------|---------|
| ✅ Transaction hash | Deployer draining LP: `0x...` |
| ✅ Contract analysis | Honeypot.is showing sell blocked |
| ✅ Social proof | Team members are fake / stock photos |
| ✅ Code analysis | Mint function allows infinite supply |
| ✅ Historical pattern | Same deployer wallet rugged 3 other projects |

Weak reports that are likely to be rejected:
- "This looks like a scam" with no evidence
- Price went down
- Team didn't respond to messages

---

## Security Signals as Evidence

Use DappScore's own [Security Signals](../signals/overview.md) as evidence in your report. For example:

- Honeypot panel showing "Honeypot detected"
- Deployer History showing 5+ previous scam contracts
- Token Distribution showing 95% held by one wallet
- Whale Tracker showing mass insider dump

Screenshot or link directly to these panels when submitting your report.
